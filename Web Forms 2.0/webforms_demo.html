2. Built-in Validation Attributes

HTML5 provides powerful client-side validation without requiring JavaScript.

Validation Attributes Demo

This field cannot be empty

Must be exactly 5 digits

Between 8 and 20 characters

Must be at least 18

Separate multiple emails with commas

Code Examples:

<!-- Required Attribute --> <input type="text" required> <!-- Pattern Validation (Regular Expression) --> <input type="text" pattern="[0-9]{5}" title="Please enter a 5-digit ZIP code"> <!-- Min/Max Length --> <input type="password" minlength="8" maxlength="20"> <!-- Min/Max Values --> <input type="number" min="18" max="120"> <!-- Multiple Values --> <input type="email" multiple>

Custom Validation with JavaScript

Password Confirmation Example

// Custom Validation with JavaScript <script> const password = document.getElementById('newPassword'); const confirmPassword = document.getElementById('confirmPassword'); confirmPassword.addEventListener('input', function() { if (password.value !== confirmPassword.value) { confirmPassword.setCustomValidity('Passwords do not match'); } else { confirmPassword.setCustomValidity(''); } }); </script>

4. The <datalist> Element

The datalist element provides autocomplete suggestions for input fields.

Live Example: Browser Selection

Type to see suggestions or select from the dropdown

HTML Code:

<label for="browser">Choose your browser:</label> <input list="browsers" id="browser" name="browser"> <datalist id="browsers"> <option value="Chrome"> <option value="Firefox"> <option value="Safari"> <option value="Edge"> <option value="Opera"> </datalist>

✓ Datalist Features

  • Users can type custom values or select from suggestions
  • Suggestions appear as the user types
  • No JavaScript required for basic functionality
  • Works with various input types

5. The <progress> Element

The progress element displays task completion progress.

Live Examples

Upload Progress: 45%
45%
Download Progress: 75%
75%
Processing (Indeterminate):

No value attribute = indeterminate state

HTML Code:

<!-- Determinate Progress --> <label for="fileProgress">Upload Progress:</label> <progress id="fileProgress" value="45" max="100">45%</progress> <!-- Indeterminate Progress (no value) --> <label>Processing...</label> <progress></progress>

JavaScript Example (Dynamic Update):

// Simulating file upload progress let progress = 0; const progressBar = document.getElementById('fileProgress'); const interval = setInterval(() => { progress += 5; progressBar.value = progress; if (progress >= 100) { clearInterval(interval); } }, 200);

6. The <meter> Element

The meter element represents a scalar measurement within a known range.

Live Examples

15%

Red (critically low)

HTML Code:

<label for="diskSpace">Disk Usage:</label> <meter id="diskSpace" value="75" min="0" max="100" low="40" high="80" optimum="20">75%</meter>

Meter Attributes Explained:

Attribute Description Example
value Current measurement 75
min Minimum value (default: 0) 0
max Maximum value (default: 1) 100
low Lower bound of normal range 40
high Upper bound of normal range 80
optimum Optimal value (affects color) 20

✓ Color Coding Logic

  • Green: Value is in the optimal range
  • Yellow/Orange: Value is suboptimal but acceptable
  • Red: Value is in a problematic range
  • The optimum value determines which direction is "better"

7. Progress vs Meter Comparison

Feature <progress> <meter>
Purpose Task completion progress Scalar measurement within range
Use Case File uploads, loading indicators, installation progress Disk usage, battery level, test scores, ratings
Direction Always increasing towards 100% Can fluctuate up or down
Color Coding No automatic color indicators Green/yellow/red based on value and ranges
Indeterminate State Yes (no value attribute) No
Typical Range 0 to max (usually 100) min to max with low/high/optimum

When to Use Each:

Use <progress> for:

  • File upload/download progress
  • Installation or setup progress
  • Form completion percentage
  • Loading indicators
  • Step-by-step wizards

Use <meter> for:

  • Disk space usage
  • Battery level indicators
  • Test scores or grades
  • Temperature readings
  • Rating systems
  • Capacity indicators
